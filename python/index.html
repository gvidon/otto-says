Строки

Везде для строк используются одинарные кавычки.

Для конкатенации строк когда это возможно следует использовать шаблоны:

'http://mywebsite.com/%u/%d' % (username, id)

Длина строки кода

До 132-х символов.

Комментарии

Все комментарии пишутся с большой буквы, с одним пробелом после #. Блок/строка с комментарием отделяется от следующего блока/строки, к которому не относится комментарий пустой строкой.

Строка кода с комментарием на той же строке не отделяется от остального кода пустой строкой

Комментарий более чем на две строки заключается в '''

Параметры лямбда-функций

Параметры лямбда-функций должны называться заглавными литерами или парой литер. Чтобы сохранить их компактность и предотвратить избыточность действий в инлайн функциях.

lambda P: P.upper()

Вертикальное выравнивание

К содержимому списка/дикта и списку импорта применяется вертикальное выравнивание значений, если в списке больше 3-х элементов. Например:

import os
import sys

from django        import settings
from django.models import Model
from django.forms  import Form
from selenium      import WebDriver

from myapp.app import views
from myapp.util import randomizer
from myapp import models

Выравнивание применяется к последовательности строк, не содержащей пустую строку.

Списки параметров и значений

Если параметры функции/метода или значения списка/дикта не удается разместить на одной строке следует перечислить каждый на отдельной, разместив закрывающую скобку на следующей после последнего элемента строке:

user = User.objects.filter(
	is_confirmed=True,
	gender='male',
	age__gt=25,
).exclude(is_online=False)

Последняя запятая

В списках параметров функиций/методов и значений списков/диктов где каждое значение размещено на новой строке последний элемент на последней строке должен завершаться запятой.
